## ChatFix-UX-Accessibility-Trust
- **Mejoras de Accesibilidad y Confianza UI**: Implementadas mejoras en contraste de `:focus-visible` a 3px globales, reestructurados los banners de error `.notice.error` usando `aria-live="assertive" ` para la pantalla de Auth Failure, corregidos conflictos con los nombres de clases CSS de los spinners (`.largeSpinner`), y aumentados los touch targets de inputs móviles (`.passwordToggleBtn`) a 44x44px min.
- **ChatFix-AI-Ops**: Operational security and validation hardened. The configuration parser `validateStartupConfig()` in `backend/index.js` was refined to eliminate duplicate URL validations that already occur via the `safeUrl` helper. The `STATUS_POLL_INTERVAL_MS` fallback logic was fixed to robustly prevent invalid inputs (non-numeric strings or values < 1000ms) from crashing the server by properly triggering a warning log and resetting the value to a safe 60000ms default. Additionally, dynamic property names (`PUT_TEMPERATURE`, `PUT_MAX_TOKENS`, `PUT_TIMEOUT_MS`) were mapped to `safeNumber` inside `PUT /api/ai/config` to log accurate warnings on invalid incoming user configurations. Finally, `docker-compose.yml` was hardened to use a strict default fallback string (`${API_KEY:-tu_contraseña_super_segura_aqui}`) for the API Key so that completely omitting the variable in Docker deployments does not unintentionally disable the core authentication mechanism. `docs/OPERATIONS_RUNBOOK.md` was updated to document these new constraints.
